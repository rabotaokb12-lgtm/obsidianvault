#termins
**Agile**- методология разработки, где выпускают продукт быстро и маленькими кусочками, получая быструю обратную связь

**Ansible, Chef, Puppet** - инструменты, что указывают какие пакеты, какие конфиги и т.д установить на сервере автоматически 

**APT/ yum/ dnf / zypper** - пакетные менеджеры для разных семейств Linux(Ubuntu/Debian, CentOs/RedHat и т.д). С их помощью можно обновлять и устанавливать пакеты(программы).

**Bash** - язык скриптов для автоматизаций в Linux(циклы, условия, работа с файлами)

**Ci/CD** (Continuous Integration/ Continuous Delivery/Deployment) - процесс, при котором  каждое новое изменений в коде:
 -Автоматический собирается и проверяется тестами(CI).
 -Автоматический разворачивается (CD) или готовиться к развёртыванию.
 CI/CD Pipeline:
 1. Автоматически забирает код из репозитория(GitHub, GitLab. BitBucket и т.д) 
 2. Собирает приложение(build): устанавливает зависимости, собирает пакеты, выполняет необходимые скрипты
 3. Запускает тесты( юнит тесты, интеграционные и любые другие)
 4. Если тесты пройдены разворачивает(деплоит) приложение на сервер - без ручных кликов

**Devops** - совокупность культур и практик, объединяющих разработку(Dev) и эксплуатацию(Ops), чтобы быстрее и надёжнее выпускать обновления.

**Deployment** - объект, определяющий сколько копий подов нужно и обновляет их автоматический.

**Docker** - платформа, позволяющая "упаковать" приложения и все зависимости в контейнер, и там его и запускает для теста

**DockerFile** - текстовый сценарий, описывающий, как собрать контейнер(установить пакеты, скопировать код, запустить приложение)

**ELK** - стэк для логирования: 
- ElastickSearch - 
        хранилище, поисковая система и бд, 
        нужна для индексаций и хранения логов, чтобы по ним можно было быстро искать
        логи превращаются в документы
- Logstash -
        Форматирует, фильтрует и обагощает логи (например добавляет гео по IP)
        После обработки отправляет данные в ElastickSearch
- Kibana -
         это веб интерфейс в ElastickSearch
         Позволяет строить дашборды, графики, фильтры и запросы по логам.

**Fluentd** - альтернатива Lohstash, тоже собирает логи из разных источников и отправляет их в базу (ElastickSearch и т.д)

**GitHub Actions** - платформа авто-сборки и деплоя, встроенная в GitHub

**GitLab** - платформа для разработки, хранения и автоматизаций кода, построенная вокруг системы контроля Git.

**Gitlab CI** - CI/CD встроенная в GitLab

**Grafana** - рисует красивые графики и дашборды из метрик Prometheus(или др источников)

**Jenkins** - один из первых популярных серверов для  CI/CD

**Kubernetes** - оркестратор, который автоматический распределяет контейнеры по серверам, перезапускает упавшие и масштабирует, когда растёт нагрузка.

**Pod** - минимальная единица в Kubernetes, внутри которой запускается контейнер.

**Prometheus** - инструмент, собирающий "метрики"(CPU, память, кол-во запросов) чтобы DevOps видел состояние сервиса в реальном времени.

**Playbook** - файл формата YAML, который описывает: Что нужно делать, На каких серверах, В каком порядке

**Registry** - хранилище Docker-образов(например DockerHub). Позволяет "пушить" образы а потом "пулить" их на другой сервак.
Push - загрузить образ в облако
Pull - скачать оттуда образ обратно

**Service** - позволяет объединить поды в единый "сетевой сервис" с балансировкой

**SNMP(Simple Network Management Protocol)** - протокол для мониторинга сетевого оборудования(роутеров, свитчей, принтеров и т.д), передаёт базовые метрики(трафик, состояние)

**SSH(Secure Shell)** - способ безопасно подключаться к серверу через командную строку. Позволяется DevOps зайти на удалённую машину и управлять ею.
SSH ключи - это пара:
                Приватный ключ(у себя)
                Публичный ключи(отдаётся серверу)
Нужно это для безопасного подключения к удалённым серверам, репозиторям(GitHub и т.д) или другим системам без ввода пароля каждый раз

**SSL-сертификат** - необходим, чтобы шифровать трафик и убрать предупреждение "недоверенный сайт"

**Systemd** - механизм управления службами(демонами) в современных Linux. Позволяет запускать, останавливать, и автоматически поднимать сервисы после перезагрузки.

**IaC** - это Infrastructure as Code, это подход при котором инфраструктура (сервера, сети, бд, хранилища и т.д) описываются в виде кода и управляются так же, как обычное программное обеспечение. 

**Terraform** - IaC инструмент, создающий ресурсы в облаке(виртуалки, сети) на основе файлов .tf. Позволяет "программировать" структуру.

**Zabbix** - классическая система мониторинга. Умеет собирать метрики через агенты или SNMP, хранить их и отправлять алёрты.

**Планировщик фич** - DevOps заранее спрашивает, как тестировать,, какие ресурсы понадобятся, нужен ли автоскейлинг, чтобы не возникло сюрпризов накануне релиза. 

**Скоординированные релизы** - слаженная работа Devops с QA, разработчиками, менеджерами, безопасниками: все знают, когда и как будет выкатываться новая версия.

**PipeLine** - последовательность этапов обработки данных или задач, где результат одного шага автоматом передаётся следующиму
Pipeline:
  - Скачивание кода из Git
  - Сборка проекта
  - Прогон юнит-тестов
  - Проверка качества кода (линтеры)
  - Деплой на сервер

**Устанавливать зависимости** - подключение нужных библиотек для запуска и работы проекта
**Сборка пакета** - подготовка проекта в виде готового архива или установщика, для дальнейшей установки и запуска где ни будь.  

**Юнит тест** - проверка отдельной части кода(функция, метод) в изоляций, не зависит от бд, сети и др моделей.
**Интеграционный тест** - проверяет как разные части системы работают вместе. Пример контроллер - сервис - бд

**Кластер** - в it среде это несколько серверов, соеденённых в одну систему
Один сервак отвечает за - обработку логики
Второй - БД
Третий - Интерфейс
И т.д
