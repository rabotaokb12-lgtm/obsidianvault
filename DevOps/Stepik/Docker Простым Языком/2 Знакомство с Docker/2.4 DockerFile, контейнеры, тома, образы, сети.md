### 1. Образы(Images)
**Образ** - это шаблон контейнера:
- Базовая файловая система(ubuntu. alpine и т.д)
- Установленные программы и библиотеки 
- Набор инструкций(какой порт открыть, какой процесс запустить)
При запуске контейнера, Docker фактически берёт образ и оживляет его:
- Образы хранятся в локальном кэше на вашей машине
- Если нужного образа нет, **Docker Daemon** скачивает его из **реестра**(Docker Hub)
- Образы состоят из слоёв `layered filesystem`: чтобы изменить один слой, не нужно перестраивать все остальные
### 2. Контейнеры
**Контейнер** - запущенный экземпляр образа. В момент запуска
- Docker создаёт *namespace*(процессы, сети, монтирование)
- Создаётся слоя Copy-on-Write
	 В этой слой и попадают все изменения в контейнере
	 Это нужно чтобы не портить исходный образ
- Запускается основной процесс(nginx, python app.py)
### 3. Сети
Каждый контейнер по умолчанию получает свой изолированный сетевой стек: 
- Контейнер имеет свой внутренний IP
- Общение между контейнерами организуется через вирт.сети, который *Daemon* настраивается автоматически
- Можно пробросить порты хоста внутрь контейнера(-p 8080:80), чтобы был доступ извне
Основные типы сетей:
- **brigde** - мост, дефолтный
- **host** - контейнер использует сеть хоста напрямую
- **none**
- **overlay** - для кластера Docker Swarm, когда контейнеры разбросаны по разным хостам.

### 4. Тома(Volumes) и хранение данных
По умолчанию, файлы внутри контейнера живут только в слое Copy on Write, и пропадают когда контейнер удалён. Для сохранение данных дольше:
- **Volumes** - специальные тома, который существуют вне контейнера но монтируются внутрь.
- **Bind mounts** - монтирование директории хоста прямо в контейнер
Это позволяет:
- Сохранять данные между перезапусками контейнеров
- Обмениваться файлами между контейнерами и хостом 
### 5. DockerFile, docker-compose, registry
- **DockerFile**: это шаблон для создания образа
- **docker-compose**: инструмент для описания (в Yaml) нескольких сервисов, сетей и томов. 
- **Registry(реестр)**: хранилище образов(Docker Hub, GitHub Container Registry, локальный )
- **Docker CLI** - набор команд(run, ps, build, pull, push)
