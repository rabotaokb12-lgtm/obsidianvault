`docker run имя образа` - запускает образ с контейнером, если такого образа локально нет то скачивает из Docker Hub
`docker run --name mynginx -d -p 8080:80 nginx`
	--name имя: дать контейнеру понятное имя
	-d: запускает контейнер в фоновом режиме
	-p 8080:80 - Привяжи порт 8080 на *хосте* к порту 80 *внутри контейнера*
`docker stop mynginx` - остановить контейнер
`docker rm mynginx` - удалить контейнер
`docker ps` - список запущенных контейнеров 
`docker ps -a` - список всех контейнеров(даже остановленных)
`docker images` - список локальных образов
`docker restart имя контейнера` - перезапуск контейнера
`docker start имя контейнера` - запустить контейнер 
`sudo systemctl restart docker` - перезапуск самого докер

Ещё команды 
`docker run -d -e ENV=prod -p 8080:80 myimage`
	-e ENV=prod: переменная среда которая может использоваться приложением
		Здесь `ENV=prod` может сигнализировать приложению, что оно запущено в «production»-режиме.
`docker run --rm имя_образа` - запустит контейнер и сразу же его удалит 

`docker pull имя` - скачивает образ из реестра 
	docker pull имя:14 - :14 это тег
`docker rmi имя:мб тег` - удаление образа из локалки
`docker kill` - УБИВАЕТ НАХРЕН КОНТЕЙНЕР
	- В отличий от docker stop который отправляет сперва *SIGTERM* он отправляет *SIGKILL* 
		- *SIGTERM* - мягкое завершение, сохранение данных корректное завершение и т.д
		- *SIGKILL* - УБИВАТЬ УБИВАТЬ УБИВАТЬ УБИВАТЬ
`docker rename old_name new_name` - переименовать контейнер
`docker port name_container` - какие порты слушает контейнер
`docker inspect name_container` - подробная инфа в формате json(Сеть, тома, переменные окружения и т.д)
`docker logs name` - выводит логи контейнера 
	`-f` - флаг что даёт мониторинг логов в реал тайме
