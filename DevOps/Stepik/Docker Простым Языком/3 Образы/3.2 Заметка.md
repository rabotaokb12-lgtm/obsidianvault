В файл Dockerfile пишем
```nginx
FROM alpine:3.17 # Базовый образ
COPY entrypoint.sh / # Копируем наш скрипт в образ
CMD ["sh", "/entrypoint.sh"] # Команда, которая запустится в контейнере
```

`docker build -t my-first-image .`
	docker build - создаёт image
	-t - образует имя 
	. - ищи Dockerfile в этой же директорий 
`docker run --rm my-first-image` - --rm типо удалить сразу контейнер как он остановиться 

## Подробнее про слои 
### 1. FROM - базовый слой 
`FROM image[:tag]` 
	Тут пишется какую систему накатить прим: 
	`FROM alpine:3.17` 
	`tag` - туда можно вписать версию и ещё чота хз, если не прописывать тэг то Docker возьмёт `:latest`, если он есть
	`FROM scratch` - Пустой базовый образ, исключение, используется для минимальный бинарников(напр. GO)
		Бинарник это уже собранная программа готовая к запуску, не требует компилятора
### 2. RUN - исполнение команд в процессе сборки 
`RUN команда` - в основном установка пакетов
	каждый RUN создаёт новый слой в образе
	Прим: `RUN apt-get update && apt-get install -y curl`
**Оптимизация:** старайтесь объединять команды, чтобы не плодить лишние слои. Пример:
```swift
RUN apt-get update && \
    apt-get install -y curl && \
    rm -rf /var/lib/apt/lists/*
```
### 3 COPY и ADD - перенос файлов в образ
#### 3.1 COPY
`COPY локальный_путь путь_в_контейнере` - перенос файлов из хоста в образ 
Если надо копировать _только_ определённые файлы (исключая node_modules и прочее), используйте `.dockerignore`.
**Пример:**
```bash
COPY package.json /app/
RUN npm install
COPY . /app
```
Сначала копируем только `package.json` (чтобы не перегенерировать слой при любом изменении кода), ставим зависимости, а потом копируем всё остальное.
#### 3.2 ADD 
`ADD локальный_или_URL путь_в_контейнере` 
	Отличие в том что можно распаковать **tar-архив** или скачать файл по **URL**
	`ADD name.tar.gz /путь в контейнера` - автоматом распокует
### 4 CMD и ENTERPOINT - определяем что запускать 
`ENTRYPOINT ["python", "myapp.py"]` - указывает что именно по итогу нужно запустить 
	Аргументы из *docker run* по типу `--port 8080` так же идут туда: `python app.py --port 8080` 
`CMD ["--port", "8080"]` - передаёт аргмуенты в ENTRYPOINT ИМЕННО В ДАННОМ СЛУЧАЕ в связке с ENTRYPOINT
	Так же в CMD может передаться другая команда из *docker run*: `docker run -it myapp /bash/sh` - передаться в cmd и оно запустит именно это

**CMD** так же можно использовать отдельно 
`CMD ["npm", "start"]` - тут она запускает файл Node js
	Объяснение допустим есть какой то файл Node js и вот в этом файле есть строчки:
	{
	  "name": "my-app",
	  "scripts": {
	    "start": "node server.js"
	  }
	}
	И вот **npm** - читает *scripts* а **start** - аргумент в скриптс 
	Таким образом получается что эта команда просто запускает этот node.js файл 
### 5. ENV - задаём переменное окружения
`ENV ключ значения` - ну ты помнишь что за переменное окружения
	Прим: `ENV NODE_ENV=production` 
	**Docker** с помощью ENV создаёт локальную переменную в линуксе и уже код в программе должен брать эту переменную оттуда
	Вот в чём смысл
### 6. EXPOSE - заявление о порте 
`EXPOSE port` - такой то порт слушает контейнер но одих хуй нужен docker run -p. Так как этот порт не может сам по себе наружу выйти из контейнера

### Дополнительно: `WORKDIR`, `USER` и другие инструкции
- `WORKDIR /app`: переключает рабочую директорию. Все последующие `RUN/CMD/ENTRYPOINT` выполняются в `/app`.
- `USER node`: меняет пользователя, от которого будут выполняться команды (безопасность: не оставаться `root`).
- `VOLUME /data:` указывает, что `/data` является точкой монтирования тома (volumes). Но это мы разберём позже.
## Пример
```coffeescript
FROM alpine:3.17

# Устанавливаем нужные пакеты (слой RUN)
RUN apk add --no-cache nodejs npm

# Меняем рабочую директорию
WORKDIR /app

# Копируем package.json и устанавливаем зависимости
COPY package*.json ./
RUN npm install

# Копируем остальной код
COPY . .

# Задаём переменные окружения
ENV NODE_ENV=production

# Указываем порт (декларативно)
EXPOSE 3000

# Команда по умолчанию
CMD ["npm", "start"]
```

- Собираем: `docker build -t my-node-app .`
- Запускаем: `docker run -p 3000:3000 my-node-app`
