## 1. ARG 
#ARG
Если переменная нужна только на этапе СБОРКИ. 
Прим: версия пакета, секретный ключ
Всё это не остаётся в контейнере оно стирается после сборки образа
`docker build --build-arg имя_переменной=значение` 
#--buid-arg

**dockerfile**:
`ARG MY_PACKAGE_VERSION=1.2.3'
`RUN pip install my package==$MY_PACKAGE_VERSION`
## 2. LABEL
#LABEL
Используется для оставление инфы в метаданных контейнера
Прим: кто автор, когда была сделано,заметка и т.д
Кста эти данные так же могут использоваться дальше в CI/CD
``` dockerfile
LABEL maintainer="kotak@gmail.com" \
	  version="0.0.1" \
	  description="poshel_nahui"
```
*просмотр метаданных*:
`docker image inspect имя_образа` 
## 3. USER
#USER
Все команды в контейнере обычно выполняются через **root**, если нужно выполнить через другого пользователя то вот тогда используется(выполняются обычно команды **entrypoint** и **CMD**)
`RUN useradd -D имя_юзера` - создание пользователя 
`-D` - без пароля 
`adduser` - в alpine
**dockerfile**:
```dockerfile
RUN adduser -D myuser
USER myuser
```
## 4. VOLUME
#VOLUME
Используется для того чтобы вывести какие то данные за пределы контейнера
Прим: логи 
`VOLUME /путь`
## 5. ONBUILD 
#ONBUILD
Родительский образ, в образе **B** в строке **FROM** можно указать образ **А** и в образе **B** выполняться команды из образа **A** 
**образ А**
``` dockerfile
FROM node:16
ONBUILD COPY package*.json /app/
ONBUILD RUN npm install
```
**образ B**
```dockerfile
FROM your-node-base:latest
COPY . /app
CMD ["npm", "start"]
```
## 6. STOPSIGNAL
#STOPSIGNAL
При остановке контейнера`docker stop` **Docker** отправляет **SIGTERM** главному процессу. С помощью **STOPSIGNAL** можно поставить что то другое
Что можно поставить 

| Сигнал  | Номер | Что означает            |
| ------- | ----- | ----------------------- |
| SIGTERM | 15    | Корректно завершиться   |
| SIGKILL | 9     | Немедленно убить        |
| SIGINT  | 2     | Прерывание (Ctrl+C)     |
| SIGHUP  | 1     | Перезапуск / reload     |
| SIGQUIT | 3     | Завершение с дампом     |
| SIGUSR1 | 10    | Пользовательский сигнал |
| SIGUSR2 | 12    | Пользовательский сигнал |
| SIGSTOP | 19    | Пауза процесса          |
| SIGCONT | 18    | Продолжить после паузы  |
`STOPSIGNAL SIGQUIT`
## 7. HEALTHCHECK
#HEALTHCHECK
Сигнал жив ли контейнер, будет показываться командной 
`docker ps` в столбце **STATUS** будет *healthy*/*unhealthe*
Вообще есть много разных способов проверки но тут будет отправляться HTTP запрос с промежутками 
```dockerfile
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD curl -f http://localhost/health || exit 1
```
## 8. SHELL 
#SHELL 
По умолчанию linux docker контейнер использует `sh -c` оболочку
Вот команда нужно чтобы поменять shell 
```dockerfile
SHELL ["/bin/bash", "-c"]
```
