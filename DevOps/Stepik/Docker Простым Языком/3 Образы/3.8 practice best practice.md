### GO fix
#### main.go
``` go
package main

import (
    "fmt"
    "log"
    "net/http"
    "os"
)

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hello from Go Docker container!")
}

func main() {
    port := os.Getenv("PORT")
    if port == "" {
        port = "8080"
    }

    http.HandleFunc("/", handler)
    log.Printf("Starting server on port %s...", port)
    log.Fatal(http.ListenAndServe(":"+port, nil))
}
```
#### dockerfile
``` dockerfile
FROM golang:1.19-alpine
WORKDIR /app
ENV PORT=8080
COPY main.go .
RUN go build -o app main.go
EXPOSE 8080
CMD ["./app"]
```
## Node nginx
```dockerfile
FROM node:16 AS build-stage
WORKDIR /usr/src/app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build-stage /usr/src/app/dist /usr/share/nginx/html
EXPOSE 8000
CMD ["nginx", "-g", "daemon off;"]
```
CMD ["nginx", "-g", "daemon off;"] - важный момент это запуск с параметрами команда выглядит так nginx -g daemon off 
Прикол в том что контейнер живёт только пока живёт главный процесс 
А NGINX уходит после запуска в backgroung(процесс уходит в фон(типа контейнер бы думал что главный процесс умер и ушёл бы тоже в смерть))
Вот команда это переводит nginx с background в foreground(процесс работает прямо в терминале)
