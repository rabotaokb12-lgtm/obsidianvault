**Система контроля версий(Version Control System, VCS)** решает:
- **Проблема потери данных** - любое изменение сохраняется навсегда
- **Проблема командной работы** - Когда несколько разрабов изменяют одни и те же файлы, VCS автоматом отслеживает все изменения и предоставляет механизмы для их объединения 
- **Проблема аудита изменений** - Каждый модификация связана с конкретным автором и имеет описание
- **Проблема экспериментальной разработки** - VSC позволяет создавать изолированные ветки
### Архитектурные подходы
- **Локальные системы VCS** - хранят всю инфу на одном компе(прим. RCS(Revision Control System)). 
- **Централизованные системы(CVCS)** - Subversion(SVN) или Perforce. От сервера дают выдаётся актуальная версия всем остальным
- **Распределённые системы(DVCS)** - Git или Mercurial
  Полная копия репозитория на каждом клиенте
  Большинство операций выполняются локально потом при необходимости синхронизируется со всеми остальными
Git использует криптографически стойкую хеш-функцию SHA-1 для идентификации и проверки целостности всех объектов в репозитории. Это означает, что любое изменение в истории будет немедленно обнаружено
