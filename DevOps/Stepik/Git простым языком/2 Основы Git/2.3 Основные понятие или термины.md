### Репозиторий
Это место где **Git** хранит всю инфу о проекте 
Физический это папка проекта в которой создана скрытая директория `.git` 
```cmake
project/
├── .git/          # Служебная папка Git
│   ├── objects/   # Хранилище всех версий файлов
│   ├── refs/      # Информация о ветках
│   ├── HEAD       # Указатель текущего положения
│   └── config     # Настройки репозитория
├── src/           # Файлы проекта
├── docs/          # Файлы проекта
└── README.md      # Файлы проекта
```

### Коммит
Зафиксированное состояние проекта в определённый момент времени. Он содержит:
- Полный снимок всех файлов проекта на момент создания коммита 
- Уникальный идентификатор(хеш)
- Инфа об авторе и времени создания 
- Описание изменений 
- Ссылка на пред. коммит(родитель)
Пример 
	commit a5f3b8c7d9e2f1a4b6c8d0e3f5a7b9c1d3e5f7a9
	Author: Anna Petrova <anna@example.com>
	Date:   Mon Mar 15 10:30:00 2024 +0300
    Исправлена ошибка валидации в форме регистрации

### Ветка 
Это именованный указатель на определённый коммит. Технически ветка представляет собой простой текстовый файл, содержащий идентификатор коммита
Основные характеристики веток:
- Создание веток не копирует файлы или историю
- При создании нового коммита ветка автоматом передвигает в него 
- Неограничено количество веток 
- Переключение между ветками изменяет содержимое рабочей папки 
Стандартная ветка `main` или `master`
`feature` - доп ветки создают для разработки новых функций и т.д 
**HEAD** - текущее положение в репозиторий. Показывает активную ветка которая указывает на последний коммит
```css
HEAD → main → a5f3b8c
```

Типичный процесс работы:
1. В репозитории есть ветка `main`, указывающая на последний коммит
2. Создаётся новая ветка `feature` для разработки новой функции
3. В ветке `feature` создаются новые коммиты
4. Ветка `feature` автоматически передвигается на новые коммиты
5. Ветка `main` остаётся на прежнем месте
6. После завершения работы изменения из `feature` вливаются в `main`
