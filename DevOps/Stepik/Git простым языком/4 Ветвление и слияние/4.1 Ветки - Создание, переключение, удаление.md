### 1. Cоздание git branch, git switch -c
`git branch name` - создать ветку
`git switch -c name` - создать ветку и переключиться на неё

Имя ставиться обычно такое:
**release/**
**bugfix/**
**feature/**
**hotfix/** 
### 2. Переключение git switch
`git switch name` - переключиться на ветку
`git switch -` - переключение на пред. ветку 
### 3. Инфа о ветке git branch
`git branch` - покажет в какой ты ветке
`-v` - покажет последний коммит ветка
`--merged` и `--no-merged` - покажет статус слияние с текущей веткой
`-a` - покажет и локальные и удалённые ветки 
### 4. Удаление ветки 
`git branch -d` - удалит ветку только если она полностью слита с текущей или *upstream* - веткой 
`-D` - принудительное удаление

### Полезная команда
`git log --graph --oneline` - наглядное представление истории веток
