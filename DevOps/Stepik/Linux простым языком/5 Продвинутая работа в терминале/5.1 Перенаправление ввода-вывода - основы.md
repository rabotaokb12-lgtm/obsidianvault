### Стандартные потоки ввода-вывода
- **STDIN(0)** - данные которые поступают на вход команде(обычно с клавы)
- **STDOUT(1)** - данные которые выводит команда при успешном выполнений
- **STDERR(2)** - ошибки и предупреждения, выводимые командой
#### Символы перенаправления
`>` - перезапись STDOUT в файл
	Прим: `ls -l > files.txt` - запись результата в файл
`>>` - запись STDOUT в конец файла
	Прим: `ls -l >> files.txt` 
`2>` - запись STDERR в файл
	Прим: `ls /нет такой папки 2> errors.txt`
`2>&1` или `&>` - запись STDOUT и STDERR в один файл
	Прим: `ls this_dir /нетупапки > text.txt 2>&1`
		   `ls this_dir /нетупапки &> text.txt`
`<` - использование файла в качестве STDIN
	Прим: `sort < text.txt` - выведет на экран отсортированные строки файла
`|` - конвеер(pipes) соединяет несколько команд, вывод одной команды становиться входом следующей
	Прим: `ls -l | less` - покажет вывод постранично 
#### Расширенные команды
`команда > output.txt 2> error.txt` - Вывод STDOUT в 1 файл а STDERR во 2 файл 
	Прим: `ls this_dir /нетпапки > output.txt 2> error.txtx` 
`команда 2>/dev/null` - не показывает ошибки 
`команда | tee file.txt` - вывод команды одновременно на экран и запись в файл 
`tee -a` - запись в конец файла
`command << EOF > filename(в конце ещё допиши EOF)` - передача нескольких строк на вход команде 
	Пример:
	```bash
	cat << EOF > newfile.txt
	Hello, world!
	This is a multiline text.
	EOF``` 

Пример команд: 
`ls /home/usern/test_std/ /nokia | tee output2.txt 2> errobe2r.txt` - запишет результат и ошибки в 2 разных файла и покажет ещё на экране 
`cat words.txt | sed 's/$/ - Linux/' | sort` - sed заменяет что то на что то, как я понял он тут заменят конец каждой строки /$/ на - Linux
