 ### Linux - ядро, ос #linux
Ядро - сердце системы, связывает железо и программы которые ты запускаешь
Пример:
1. Ты нажимаешь сохранить
2. Программа говорит ядру сохрани
3. Ядро общается с драйверами и те записывают данные
Кароче **ядро-посредник между программами и железом** #ядро
### Архитектура Linux 
1. **Ядро** - отвечает за управление процессами, управление ресурсами и общение процессов между собой: #ядро
	- процессами(что и когда выполняется)
	- памятью(кому сколько выделить)
	- устройствами ввода-вывода(диск, сеть, видео, звук)
	- взаимодействиями между процессами(IPC)
2. **Системные библиотеки** - набор функций которые используют другие программы, чтобы напрямую не общаться с ядром. Например: 
	- библиотека glibc(GNU C Library) даёт программам стандартные функций -читать/записывать файлы, выделять память, запускать процессы и т.д
	- Можно сказать библиотеки это переводчик между программами и ядром
3. **Системные утилиты** - отдельные программы
4. **Оболочка терминала**
5. **Графический интерфейс**

Как всё связано: Пользователь -> Системная утилита -> Системные библиотеки -> Ядро -> Железо 
### Процессы 
**Каждая запущенная программа - процесс** 
Процесс имеет:
* PID - уникальные идентификатор процесса 
* Свой кусок памяти
* Доступ к ресурсами

У каждого процесса есть родитель - процесс-создатель 
Самый главный процесс: *init*(более старая) или ==systemd==(более новая ), этот процесс запускается сразу после ядра
Они уже создают остальные процессы а те свои процессы, типо иерархия(дерево процессов)

Типы процессов:
* Обычные программы
* Фоновые задачи
* Демоны
### Файловая структура
"ВСЁ ЕСТЬ ФАЙЛ"
==/bin /sbin== - основные исполняемые файлы
==/etc== - конфиги
==/home== - домащние директорий юзеров
==/var== - логи, бд
==/usr== - программы в системе 
==/lib== - системные библиотеки 
==/dev== - файлы устройств
==/boot== - файлы загрузчика и ядра
==/proc==  ==/sys== - виртуальные файловые системы
