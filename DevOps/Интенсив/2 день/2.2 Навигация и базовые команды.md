# Команды для навигации в Linux
## cd — навигация по каталогам
`pwd` — показать текущий каталог  
`cd /var/log` — перейти в каталог  
`cd ~` — домашний каталог  
`cd ~username` — домашний каталог другого пользователя  
`cd -` — предыдущий каталог  
`cd ..` — на уровень вверх  
`cd ../..` — на два уровня  
`cd "Program Files"` — каталог с пробелами
## ls — просмотр содержимого каталогов
`ls -l` — подробный вывод  
`ls -ld` - чекнуть права директорий
`ls -a` — скрытые файлы  
`ls -la` — подробный + скрытые  
`ls -lh` — читаемые размеры  
`ls -i` — inode номера  
**inode** — идентификатор файла, где хранятся метаданные (размер, даты, права). #inode
## tree — структура каталогов
Установка: `sudo apt install tree`  
`tree -L 2` — глубина  
`tree -d` — только каталоги  
`tree -h` — размеры  
`tree -p` — права

# 📁 Работа с файлами и каталогами
## mkdir — создание каталогов
`mkdir newdir`  
`mkdir -p /a/b/c` — вложенные  
`mkdir -m 755 dir` — с правами
## touch — создание файлов
`touch file.txt` — пустой файл
## cp — копирование
`cp source dest` — копировать файл  
`cp file.txt /path/` — в директорию  
`cp -r dir1 dir2` — рекурсивно(папку и всё что внутри)  
`cp -l file hardlink` — жёсткая ссылка  
`cp -s file symlink` — символическая  
`cp -a src dst` — копировать с правами, датами, владельцем
``cp -v`` - показывает процесс копирования 
``cp -i`` - спрашивает подтверждение перед записью
## mv — перемещение и переименование
`mv old new` — переименовать  
`mv file /path` — переместить файл  
`mv dir1 dir2` — переместить папку
## rm — удаление
`rm file` — удалить файл  
`rm file1 file2` — несколько  
`rm -r dir/` — рекурсивно папку  
`rm -f file` — принудительно  
`rm -rf dir/` — очень опасно

# 📜 Работа с содержимым файлов
## cat — вывод содержимого
`cat file.txt`  
`cat file1 file2` — вывести подряд  
`cat -n file` — нумерация строк
## less — постраничный просмотр
`less file.txt`  
Space — вниз  
b — вверх  
g — начало  
G — конец  
/текст — поиск вперёд  
?текст — назад  
n / N — следующее / предыдущее совпадение  
q — выход
## head — начало файла
`head file.txt` — первые 10 строк  
`head -n 20 file.txt` — первые N строк
``head -v`` - всегда показывает имя файла 
``head -r`` - выводит строки в обратном порядке 
## tail — конец файла
`tail file.txt` — последние 10 строк  
`tail -n 20 file.txt` — последние N  
`tail -f file` — следить в реальном времени
# 🔍 Поиск файлов и текста
## find — поиск в файловой системе
**find где искать условия действия** 
`find /path -name "filename"` — поиск по имени  
`find /path -iname "FileName"` — без регистра  
`find /path -name "*.log"` — по расширению  
`find /path -name "file.*"` — по названию без расширения
`find /path -type d -name "dirname"` — только каталоги  
`find /path -type f -name "filename"` — только файлы
`find /path -size +10M "filename"` — файлы больше 10 мегабайт (есть суффиксы `k`, `M`, `G`).
`find /path -empty "filename"` — пустые файлы/каталоги (совместите с `-type f`, чтобы искать только файлы).
`find -mmin -60` - изменённые за последние 60 мин
`find -mtime -7` - изменённые за последние 7 дней
## grep — поиск текста
`grep "pattern" file`  
`grep -i "pattern"` — без регистра  
`grep "error" /var/log/*` — по каталогу

## uniq - уникальные строки
``uniq text.txt`` - выводит только уникальные строки(но сравнивает только соседние строки) 

## file - укажет какой у файла тип файла
## wc -l - количество строк 
## sort - сортировка
``sort file.txt`` - выведет отсортированный файл
``sort file.txt -o sort.txt`` - создаст отсортированный файл(-o флаг который указывает куда записать результат сортировки)
``sort -u sort.txt`` - покажет только уникальные строки (-u это uniq)
## which / whereis — поиск бинарников
`which nginx` — путь к бинарнику  
`which -a nginx` — все пути  
`whereis nginx` — бинарник + исходники + мануалы
`whereis -l` - показывает где команда ведёт поиск

Бинарники и исполняемые файлы отличие
**Исполняемый файл** — любой файл, который можно запустить (скрипт или бинарник).  
**Бинарник** — исполняемый файл, содержащий машинный код (ELF).
# ⚡ Быстрый поиск по базе
## locate — очень быстрый поиск
`locate filename.txt`  
`locate "*.conf"`  
`updatedb` - обновляет базу файлов 
`-i` - игнор регистра 
`-n число` - ограничить количество результатов
Смысл в том что locate хранит все файлы в своей дб и в отличий от find находит файлы за доли секунды 
Но нужно обновлять базу или база сама со временем обновляться будет(раз в сутки) и конечно нужно сперва установить его
