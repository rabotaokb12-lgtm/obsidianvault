# 🟩 **Ansible — что это такое?**

Ansible — это инструмент для автоматизации настройки серверов и инфраструктуры.  
Через него можно разворачивать приложения, конфиги, пакеты и обновления **сразу на сотнях серверов**, одной командой.

---
# 🟦 **Главная сущность Ansible — Playbook**

**Playbook** — это сценарий, в котором описаны задачи (tasks), которые нужно выполнить на серверах.
Пример запуска:
ansible-playbook playbook.yml
# 🟨 **Node (ноды) в Ansible**
### ✔ Контролёр
Машина, где установлен Ansible. Отсюда запускаются playbook’и.
### ✔ Managed Nodes
Сервера, на которых Ansible выполняет задачи (подключается через SSH или WinRM).

---
# 🟫 **Inventory — список хостов**
**Inventory** — файл со списком серверов, сгруппированных по ролям:
[webserver]
192.168.1.10
192.168.1.11

Это позволяет в playbook писать:
hosts: webserver

Запуск playbook с инвентори:
ansible-playbook -i inventory.ini playbook.yml
# 🟧 **Roles — структурированная логика**
**Role** — это папка со структурой:
- tasks
- handlers
- templates
- files
- vars
- defaults

Каждая роль отвечает за одну логическую настройку: nginx, docker, firewall, users и т.д.
### Зачем нужны роли:
✔ не засорять playbook  
✔ структурировать проект  
✔ переиспользовать роли на десятках/сотнях серверов  
✔ модульность — одна роль = одна задача  
✔ читабельность  
✔ масштабирование инфраструктуры  
✔ готовые роли через **Ansible Galaxy**

Роль подключается так:
roles:
  - nginx
# 🟥 **Зачем нужен Ansible?**
- Автоматическая настройка и обновление серверов
- Развёртывание приложений
- Массовая установка пакетов и патчей
- Конфигурирование Docker, Kubernetes, Nginx, Firewall
- Выкатка версий по всем нодам
- Подготовка инфраструктуры (VM, bare metal, сетевые устройства)
