##### КАК микротику дать IP #microtic_net
Сейчас будем пример с интерфейсом **eth1** то есть схема net->eth1->microtic
1. Проверяем не выключился ли этот интерфейс /interface print
   `X` → интерфейс **выключен (disabled)**
    `R` → интерфейс **работает (running)**
    Если x то прописываем /interface enable ether1
2. /ip address add address=192.168.73.55/24 interface=ether1
3. Проверяем выдался ли /ip address print
4. Пингуем этот ip в микротике в eve ng и в cmd: ping 192.168.73.55

##### КАК микротик может выдать интернет
Дать порту к которому подключается устройство ip в ДРУГОЙ подсети
/ip address add address=192.168.100.1/24 interface=ether2

**Как добавить и удалить пути к vlan-ам в микротик**
	Добавить:
		 /ip route add dst-address=192.168.20.0/24 gateway=192.168.100.3
	Удалить:
		Посмотри номера роутов /ip route print
		Потом удаляй /ip route remove 1

**Создание бэкапа микротик**
	/export file=lab-backup
	Потом его загрузка
	/import file=lab-backup.rsc
	
**Запрет пинга на микротик**
	/ip firewall filter
	add chain=input src-address=192.168.20.0/24 dst-address=192.168.100.1 protocol=icmp action=drop comment="Блокировать ping VLAN20 -> MikroTik 100.1"

**Выдача DNS**
	/ip dns set servers=8.8.8.8,8.8.4.4 allow-remote-requests=yes
	Проверка:
	/ip dns print
